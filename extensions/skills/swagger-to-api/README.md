## Features

### Automatic Detection for Swagger 2.x and OpenAPI 3.x

Automatically identifies the input spec version (Swagger 2.x or OpenAPI 3.x) and correctly extracts the base URL, security schemes, and all path/method combinations per version. No manual version flag needed — the same command handles API specs from different sources seamlessly.

### Per-Endpoint Output in agenvoy APIDocumentData Format

Converts each API endpoint into agenvoy's flat APIDocumentData format — not JSON Schema or Function Calling format — writing one JSON file per endpoint to `~/.config/agenvoy/apis/`. This eliminates format incompatibility between tools, letting agenvoy load definitions directly without secondary transformation.

### Automatic Auth and Type Inference

Maps OpenAPI Security Schemes (bearer, apiKey header/query, OAuth2) to agenvoy auth fields, and converts OpenAPI parameter types (array, object, etc.) to agenvoy-compatible types with semantic description annotations. Developers get immediately usable API definition files without writing any format conversion logic.

## CLI Reference

### Arguments

| Argument | Required | Description |
|----------|----------|-------------|
| `swagger_json_path` | No | Path to the Swagger / OpenAPI JSON file; omit if file content is already embedded in the conversation |

### Supported Spec Versions

| Version | Detection Key | Base URL Source |
|---------|---------------|-----------------|
| Swagger 2.x | `swagger` key | `schemes[0]://host + basePath` |
| OpenAPI 3.x | `openapi` key | `servers[0].url` |

### Output Files

| File | Description |
|------|-------------|
| `~/.config/agenvoy/apis/{name}.json` | One JSON file per endpoint in agenvoy APIDocumentData format |

### Auth Type Mapping

| OpenAPI Security Scheme | agenvoy auth.type |
|------------------------|-------------------|
| http bearer / oauth2 | `bearer` |
| apiKey in header | `apikey` (with header name) |
| apiKey in query | `apikey` |
| No security | auth field omitted |
